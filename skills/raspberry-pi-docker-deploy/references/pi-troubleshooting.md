# Pi Troubleshooting

Use this reference only when Raspberry Pi deployment fails or BME680 is not visible on the host.

## 1. Confirm the real boot config path

Do not trust `/boot/config.txt` blindly on newer Raspberry Pi OS images.

If `/boot/config.txt` contains a message like `DO NOT EDIT THIS FILE`, the real file is elsewhere.

Check:

```bash
ls /boot
ls /boot/firmware
sed -n '1,120p' /boot/config.txt
sed -n '1,120p' /boot/firmware/config.txt
```

For this repository, the important setting is:

```ini
dtparam=i2c_arm=on
```

It must be enabled in the real boot config file.

## 2. Confirm the boot partition is mounted

If `/boot/firmware` is empty or fake, check mounts:

```bash
findmnt -T /boot/firmware
cat /etc/fstab
sudo blkid
```

This repository already saw a broken Pi where:

- `/boot/firmware` was not mounted
- `/etc/fstab` was nearly empty
- edits went to the wrong file

Typical expected entries:

```fstab
PARTUUID=<root-partuuid>  /              ext4    defaults,noatime  0  1
PARTUUID=<boot-partuuid>  /boot/firmware vfat    defaults          0  2
```

If the boot partition is not mounted, `dtparam=i2c_arm=on` changes will not survive in the real boot config.

## 3. Confirm GPIO 2 and 3 are actually I2C

Check:

```bash
sudo pinctrl get 2
sudo pinctrl get 3
```

Expected:

- `GPIO2 = SDA1`
- `GPIO3 = SCL1`

If GPIO 2 and 3 show plain input mode, the I2C overlay is still not active.

## 4. Confirm the I2C devices on the host

Check:

```bash
ls -l /dev/i2c*
sudo i2cdetect -l
sudo i2cdetect -y 1
```

Expected for BME680 in this repository:

- `/dev/i2c-1` exists
- bus 1 is present in `i2cdetect -l`
- bus 1 shows `76` when `SDO/ADDR` is tied to GND

If only `i2c-2` exists, do not assume the sensor moved. First fix the missing GPIO I2C bus.

## 5. Confirm Docker can see the device

Check:

```bash
docker inspect eco-knock-be-embedded-server-1 --format '{{json .HostConfig.Devices}}'
```

Expected:

```json
[{"PathOnHost":"/dev/i2c-1","PathInContainer":"/dev/i2c-1","CgroupPermissions":"rwm"}]
```

If this is missing, the compose file is wrong even if the host detects the sensor.

## 6. Confirm Docker service and filesystem state after reboot

This repository already hit a case where reboot caused:

- `/` to mount read-only
- Docker daemon to stay down

Check:

```bash
mount | grep ' on / '
sudo systemctl is-active docker
docker ps
```

Expected:

- `/` mounted `rw`
- Docker `active`

If `/` is read-only, fix the underlying mount configuration before retrying deployment.
